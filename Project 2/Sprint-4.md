## 🏃 Sprint 4 — Enterprise Architecture

**Weeks 7–8 · Project 2: Event Management API**

### 🎯 Goal

Transition from traditional N-Tier architecture to Clean Architecture and build scalable APIs using CQRS and MediatR.

---

## 📚 Concepts

- Clean Architecture
- SOLID Principles
- Dependency Inversion
- CQRS Pattern
- MediatR
- Commands & Queries
- Pipeline Behaviors

---

## 📝 Features

### 🏗️ Clean Architecture

Structure the solution following Clean Architecture principles.

- Separate Domain, Application, Infrastructure and API layers
- Organize features using Feature-Based folders
- Implement Dependency Injection
- Keep the Domain independent from external libraries

### ⚡ CQRS with MediatR

Separate read and write operations.

- Commands
- Queries
- Command Handlers
- Query Handlers
- DTOs
- Mapping

### ✔️ Request Pipeline

Improve request processing.

- Validation Behavior
- Logging Behavior
- Centralized request handling

---

## ✅ Deliverable

A Clean Architecture API where every request is handled through MediatR using the CQRS pattern.
